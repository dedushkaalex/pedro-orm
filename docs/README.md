# pedro-orm — документация

Учебный мини-ORM поверх Effect-TS с поддержкой двух движков: Postgres
(через `@electric-sql/pglite`) и SQLite (через `better-sqlite3`).

Цель проекта — на маленьком, но рабочем коде разобрать ключевые
паттерны Effect-TS (`Layer`, `Context.Tag`, `Scope`, `TaggedError`,
`withSpan`) и устройство ORM-уровня поверх «сырого» драйвера БД.

## Как читать документацию

Документация разложена по трём уровням, от «что и зачем» к «как
именно сделано»:

1. **HLD** (`docs/hld/`) — архитектурный обзор. Какие компоненты есть,
   кто за что отвечает, как они общаются, какие Effect-примитивы
   используются. Здесь нет имён файлов и функций — только идеи и
   границы.
2. **LLD** (`docs/lld/`) — подробности реализации модулей: типы,
   ключевые функции, нетривиальные решения, граничные случаи.
3. **README** проекта — установка и команды (`package.json`).

Рекомендуемый порядок чтения для нового человека:

- `docs/hld/01-architecture-overview.md` — общая картина и
  мини-учебник по используемым примитивам Effect-TS
- `docs/lld/driver-runtime.md` — как устроен слой работы с БД
- `docs/lld/dialect.md` и `docs/lld/codec.md` — как мы прячем различия
  между Postgres и SQLite
- `docs/lld/errors.md` — типизированные ошибки доменного слоя
- `docs/lld/schema.md` — type-level схема таблиц (был написан раньше)

Живой пример работы всей системы лежит в `playground/lesson-1.ts`.
